## 1. Введение

Цель тестирования - проверить корректность работы Telegram-бота
Coopera и веб-интерфейса с базой данных, а также убедиться в
стабильности логики обработки задач, взаимодействия пользователя с
системой и корректности бизнес-правил.

## 2. Методы и инструменты тестирования

### 2.1. Обзор методов тестирования

**Функциональное тестирование:** проверка корректности основных функций бота и веб-интерфейса.  
**Интеграционное тестирование:** проверка взаимодействия Telegram-бота, backend-сервера и базы данных.  

### 2.2. Обзор средств тестирования

**JUnit:** для тестирования backend-логики и модульных функций.  
**Postman:** тестирование API и интеграционного взаимодействия.  
**Selenium:** базовая автоматизация тестирования веб-интерфейса.  
**JMeter:** нагрузочное тестирование.  
**Manual testing:** ручное тестирование взаимодействия через Telegram.

### 3.1. Результаты тестов - разница от прошлого отчета

* **Выполнено тестов:** 32  
* **Пройдено успешно:** 30  
* **Провалено:** 2  
* **Коэффициент успешности:** 94%  

### 3.2. Отклонения в выполнении плана тестирования

В ходе выполнения тестового плана были выявлены следующие отклонения:

#### 1. Добавление дополнительных тестов для проверки корректности статусов

* **Причина:** Обнаружены расхождения в поведении статусов задач в Telegram-боте по сравнению с ожидаемым.  
* **Действия:** В план тестирования было добавлено 5 дополнительных юнит-тестов для проверки переходов статусов и корректности отображения.  

#### 2. Отсутствие интеграционного тестирования между Telegram-ботом и backend

* **Причина:** На данный момент реализованы только функциональные и юнит-тесты отдельных функций.  
* **Рекомендация:** В будущем добавить интеграционные тесты для проверки взаимодействия бота, сервера и базы данных.

#### 3. Отсутствие тестов безопасности и нагрузки

* **Причина:** В текущем плане тестирования не предусмотрены проверки безопасности и нагрузочные сценарии.  
* **Рекомендация:** В дальнейшем включить проверки прав пользователей, обработку ошибок и сценарии высокой нагрузки для полной оценки стабильности системы.  

### 3.3. Оставшиеся тесты и исправления в плане тестирования

**Оставшиеся тесты:**

* Тесты восстановления после сбоев.  
* Широкое тестирование совместимости в разных браузерах.  

**Изменения в плане:**

* Продлить время тестирования на 3 дня.  
* Добавить автоматизацию основных регрессионных сценариев.  

## 4. Зоны тестирования

### 4.1. Зона 1 - Telegram Bot

#### 4.1.1. Цели тестирования

* Проверка корректности логики взаимодействия с пользователем.  
* Проверка обработки команд и статусов задач.  
* Проверка корректности ответов Telegram API.  

#### 4.1.2. Критерии оценивания

* Успешное прохождение всех функциональных тестов.  
* Время ответа бота не превышает 1 секунды.  
* Отсутствие критических ошибок.  

#### 4.1.3. Результаты

* Все основные функции работают корректно.  
* Обнаружена одна ошибка в обработке статуса задачи.  
* Скорость ответа в пределах нормы.  

#### 4.1.4. Рекомендации

* Исправить найденные ошибки в логике статусов.  
* Улучшить обработку ошибок сети.  
* Добавить логирование критичных операций.  

### 4.2. Зона 2 - Web-интерфейс и база данных

#### 4.2.1. Цели тестирования

* Проверить корректность отображения задач.  
* Проверить стабильность API.  
* Оценить работу базы данных под нагрузкой.  

#### 4.2.2. Критерии оценивания

* Корректная работа всех CRUD-операций.  
* Быстрые отклики API.  
* Полная целостность данных.  

#### 4.2.3. Результаты

* Интерфейс отображает данные корректно.  
* На высоких нагрузках API снижает скорость.  
* Найдены проблемы в индексации БД.  

#### 4.2.4. Рекомендации

* Оптимизировать запросы.  
* Добавить индексы для часто используемых полей.  
* Ввести кэширование.

## 6. Обновленный план тестирования

На основании выполненного анализа и текущего состояния системы рекомендуется:  

1. **Исправление известных ошибок:**  
   * Добавление участников в команду.  

2. **Дополнительные тесты:**  
   * Юнит-тесты для проверки корректности статусов задач и бизнес-логики.  
   * Интеграционные тесты для проверки взаимодействия Telegram-бота, веб-интерфейса и базы данных.  
   * Регрессионные тесты для ключевых функций после исправления ошибок.  

3. **Дальнейшее наблюдение за стабильностью:**  
   * Мониторинг новых и устраненных дефектов до достижения точки Zero Bug Bounce.  
   * Планирование возможных корректировок в системе на основе динамики ошибок.  

4. **Автоматизация и расширение тестирования:**  
   * Включение автоматизации проверок безопасности и нагрузки в последующих циклах тестирования.
